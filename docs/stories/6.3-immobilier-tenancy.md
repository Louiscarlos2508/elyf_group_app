# Story 6.3: Immobilier Multi-tenancy & Audit Logging

## Description
This story implements full multi-tenancy and audit logging for the Immobilier module.

## Proposed Changes

### Entities
- Update `Property`, `Tenant`, `Contract`, `Payment`, and `Expense` (Immobilier) to include `enterpriseId`.
- Add `copyWith` methods for state consistency.

### Repositories
- Update `PropertyOfflineRepository`, `TenantOfflineRepository`, `ContractOfflineRepository`, etc. to map `enterpriseId`.
- Ensure all queries are scoped to the active `enterpriseId`.

### Controllers
- Integrate `AuditTrailService` into `ImmobilierController`.
- Log contract creations, payments, and property status changes.

### UI
- Ensure property management and payment screens correctly propagate the active enterprise context.

## Verification
- Unit tests for entity mapping.
- Integration tests for enterprise data isolation in Immobilier.
- Audit trail log verification.
