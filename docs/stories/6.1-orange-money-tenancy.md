# Story 6.1: Orange Money Multi-tenancy & Audit Logging

## Description
This story implements full multi-tenancy and audit logging for the Orange Money module.

## Proposed Changes

### Entities
- Update `Transaction`, `Agent`, and `LiquidityCheckpoint` to include `enterpriseId`.
- Add `copyWith` methods for state consistency.

### Repositories
- Update `TransactionOfflineRepository`, `AgentOfflineRepository`, and `LiquidityOfflineRepository` to handle `enterpriseId` in mappings.
- Ensure all queries are scoped to the active `enterpriseId`.

### Controllers
- Integrate `AuditTrailService` into `OrangeMoneyController`.
- Log cash-in, cash-out, and liquidity checkpoint events.

### UI
- Ensure forms correctly propagate the active enterprise context.

## Verification
- Unit tests for entity mapping.
- Integration tests for enterprise data isolation.
- Audit trail log verification.
