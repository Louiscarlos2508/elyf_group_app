# Story 6.2: Boutique Multi-tenancy & Audit Logging

## Description
This story implements full multi-tenancy and audit logging for the Boutique module.

## Proposed Changes

### Entities
- Update `Product`, `Sale`, `Purchase`, and `Expense` (Boutique) to include `enterpriseId`.
- Add `copyWith` methods for state consistency.

### Repositories
- Update `ProductOfflineRepository`, `SaleOfflineRepository`, etc. to map `enterpriseId` in `fromMap`/`toMap`.
- Ensure all queries are scoped to the active `enterpriseId`.

### Controllers
- Integrate `AuditTrailService` into `BoutiqueController` (or equivalent).
- Log product updates, sales, and purchases.

### UI
- Ensure POS and product management screens correctly propagate the active enterprise context.

## Verification
- Unit tests for entity mapping.
- Integration tests for enterprise data isolation.
- Audit trail log verification for Boutique operations.
