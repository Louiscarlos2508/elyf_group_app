# Brownfield UI/Frontend Enhancement

Agent workflow for enhancing existing frontend applications with new features, modernization, or design improvements. Handles existing UI analysis and safe integration.

## Steps

1. **UI Analysis** (Architect)
   - Review existing frontend application, user feedback, analytics data, and identify improvement areas.
   - Run `document-project`.

2. **PRD Creation** (PM)
   - Create `prd.md` using `brownfield-prd-tmpl`.

3. **UX Specification** (UX Expert)
   - Create UI/UX specification integrating with existing patterns.

4. **Architecture Creation** (Architect)
   - Create frontend architecture with component integration strategy and migration planning.

5. **PO Validation** (PO)
   - Validate all documents for UI integration safety and design consistency.

6. **Sharding Documents** (PO)
   - Shard documents for IDE development.

7. **Story Creation** (SM)
   - Create next story from sharded docs.

8. **Implementation** (Dev)
   - Implement approved story sequentially.

9. **QA Review** (QA)
   - Senior dev review with refactoring ability.

10. **Repeat**
    - Continue for all stories.
