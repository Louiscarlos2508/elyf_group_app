# Greenfield Service/API Development

Workflow for building new backend services and APIs from scratch.

## Steps

1. **Service Briefing** (PM)
   - Define service objectives, API requirements, and performance goals.

2. **PRD Creation** (PM)
   - Detailed requirements for API endpoints, data models, and integrations.

3. **Architecture Design** (Architect)
   - API design, schema definitions, and infrastructure planning.

4. **PO Validation** (PO)
   - Validate service technical standards.

5. **Sharding Documents** (PO)
   - Shard documents for IDE development.

6. **Story Creation** (SM)
   - Create technical stories.

7. **Implementation** (Dev)
   - Implement service and tests.

8. **QA Review** (QA)
   - Validate API contracts and performance.

9. **Repeat**
    - Continue for all stories.
